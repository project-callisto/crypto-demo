Our encryption method uses four key techniques:
  • Client-side encryption to encrypt user data within the browser
  • Oblivious PRF over an elliptic curve to prevent against brute force attacks and ensure that the key servers cannot learn the perpetrator ID and that users cannot learn Callisto’s secret key
  • Secure multi-cloud key federation to prevent Callisto’s secret key from being exposed to any one cloud provider
  • Linear Secret Sharing in order to ensure that there must be at least two victims of the same perpetrator before a user’s data can be decrypted by the Callisto Option Counselor
To begin the demo, input a perpetrator name and your name; we will refer to the combination of these inputs as “record”. This information is encrypted in the browser using the Libsodium cryptography library. We also generate a random key Krecord to symmetrically encrypt this information. (In the real product, users will be inputting an array of multiple unique perpetrator identifiers and their own verified contact information. They will also be able to edit their record with an auto-generated passphrase.)
