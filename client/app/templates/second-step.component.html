Perpetrator ID Randomization

Computed RID: {{ plainTextData.rid }}

A perpetrator ID doesn’t have enough randomness to protect itself against a brute force attack, so the first step in our design is to boost the perpetrator ID with entropy using an oblivious pseudorandom function (oPRF). The oPRF uses two inputs: a secret key from the key server and the user-inputted perpetrator ID to generate a randomized ID called RID. However, neither party wants to reveal its input to each other. Under the DDH assumption, our protocol guarantees that the user doesn’t learn the server key, and the server never learns the perpetrator ID. To protect the secret key even further, we federate it between two different cloud providers.
